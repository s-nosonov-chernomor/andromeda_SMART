Практические ориентиры для типовых линий Modbus
Открыть / скрыть
Это не жёсткие ограничения, а практические ориентиры. Если устройства медленные, нестабильные, линия длинная или шумная — выбирайте более осторожные значения и постепенно ускоряйте систему.

Основной polling

interval_ms: обычно 300–1000. Ниже — быстрее обновление, но выше нагрузка.
jitter_ms: обычно 0–100. Полезно для сглаживания пиков.
inter_request_delay_ms: обычно 0–30. Для слабых приборов полезно 5–20.
backoff_ms: обычно 100–1000.
max_errors_before_backoff: обычно 2–5.
port_retry_backoff_s: обычно 1–10 секунд.
unit_error_skip_s: обычно 0–5. Если один прибор тормозит линию — полезно 1–3.
skip_node_on_timeout: полезно включать на длинных линиях, чтобы один зависший узел не тормозил остальных.

CRC / запись / batch

crc_retry_count: обычно 0–2.
crc_retry_delay_ms: обычно 10–100 мс.
write_coalesce_ms: обычно 20–200 мс.
write_max_tasks: обычно 100–1000.
write_max_bits: обычно 8–64.
write_max_registers: обычно 4–32.
batch_read.enabled: лучше включать, если адреса плотные и идут подряд.
batch_read.max_bits: обычно 16–128.
batch_read.max_registers: обычно 8–64.

Fast Modbus

fast_modbus.enabled: включать только там, где реально нужен быстрый контур.
fast_modbus.interval_ms: обычно 20–100.
fast_modbus.jitter_ms: обычно 0–20.
fast_modbus.max_events_per_poll: обычно 50–500.
fast_modbus.bootstrap_pause_ms: обычно 10–200.
fast_modbus.rx_gap_ms: обычно 5–50.
fast_modbus.rx_hard_timeout_ms: обычно 100–1000.
fast_modbus.poll_idle_sleep_ms: обычно 50–500.
fast_modbus.poll_pending_sleep_ms: обычно 50–300.

Практические советы

Если устройства старые или медленные — увеличивайте timeout, inter_request_delay_ms и не ставьте слишком агрессивный fast modbus.
Если линия шумная — помогает crc_retry_count и небольшая crc_retry_delay_ms.
Если один прибор виснет и тормозит всю шину — включайте skip_node_on_timeout и задавайте unit_error_skip_s.
protocol_debug и test_enabled обычно включают только на время диагностики.
insecure_tls включать только для тестов и самоподписанных сертификатов.
Если после изменений стало хуже — вернитесь к более мягким значениям и увеличьте паузы между запросами.

Готовые профили настройки

Быстрые стартовые ориентиры для разных типов объектов
Открыть / скрыть
Ниже — не автоматическое применение, а готовые примеры значений. Пользователь может просто посмотреть и вручную перенести их в поля ниже, если профиль подходит под его объект.

Профиль 1. Осторожный / стабильный

Подходит для слабых приборов, длинных RS-485 линий, нестабильных адаптеров, объектов где важнее стабильность, чем скорость.
interval_ms = 800
jitter_ms = 50
backoff_ms = 500
max_errors_before_backoff = 3
inter_request_delay_ms = 15
crc_retry_count = 1
crc_retry_delay_ms = 30
unit_error_skip_s = 2.0
skip_node_on_timeout = true
batch_read.enabled = true
batch_read.max_bits = 32
batch_read.max_registers = 20
fast_modbus.enabled = false
Когда выбирать: если на линии бывают таймауты, CRC-ошибки, “подвисающие” приборы или важно минимизировать риск сбоев.

Профиль 2. Сбалансированный

Универсальный стартовый профиль для большинства нормальных объектов без явных проблем по линии.
interval_ms = 500
jitter_ms = 30
backoff_ms = 200
max_errors_before_backoff = 3
inter_request_delay_ms = 10
crc_retry_count = 1
crc_retry_delay_ms = 20–30
unit_error_skip_s = 1.0–2.0
skip_node_on_timeout = true
batch_read.enabled = true
batch_read.max_bits = 32–64
batch_read.max_registers = 20–40
fast_modbus.enabled = false или true только для специальных быстрых сценариев
Когда выбирать: если объект обычный, линия нормальная, а нужна разумная скорость без лишней агрессии.

Профиль 3. Быстрый / отзывчивый

Для коротких качественных линий, быстрых устройств и сценариев, где важна высокая частота обновления.
interval_ms = 200–300
jitter_ms = 0–20
backoff_ms = 100–200
max_errors_before_backoff = 2
inter_request_delay_ms = 0–5
crc_retry_count = 0–1
crc_retry_delay_ms = 10–20
unit_error_skip_s = 0–1
skip_node_on_timeout = true
batch_read.enabled = true
batch_read.max_bits = 64
batch_read.max_registers = 32–64
fast_modbus.enabled = true
fast_modbus.interval_ms = 20–50
Когда выбирать: если линия действительно качественная и ты уверен, что оборудование выдерживает частый опрос.

Профиль 4. Диагностика / поиск проблем

Не для постоянной эксплуатации. Используется, когда нужно понять, что происходит на линии и почему есть ошибки.
debug.enabled = true
debug.log_reads = true
debug.summary_every_s = 10–30
fast_modbus.protocol_debug = true только если реально нужен разбор fast modbus
crc_retry_count = 1–2 для проверки поведения линии
inter_request_delay_ms = 10–30 чтобы снизить шум от перегрузки
skip_node_on_timeout = true, если нужен обзор по всей линии, а не зависание на одном приборе
После диагностики такие режимы лучше выключать или ослаблять, иначе лог будет быстро расти, а нагрузка на систему станет выше.
